# Riot Games トーナメントAPIポリシー

これらを十分にお読みください。

すべてのプロジェクトはこれらのポリシーの対象となります。プロジェクトオーナーはこれらのポリシーを読み、理解することが求められます。これらのポリシーに従わない場合、Riot Games APIへのアクセスが無期限に取り消される可能性があります。

トーナメントAPIは、トーナメント主催者がコミュニティトーナメントの組織と管理を促進するために特別に作成されました。したがって、トーナメントAPIはトーナメントまたは類似のイベント専用に使用する必要があります。トーナメントAPIは、単一ゲーム用のプリメイドロビーを生成する機能のために使用すべきではありません。

トーナメントAPIに関するポリシーは非常に具体的な性質を持ち、一般ポリシーを含みますが、時には優先されることがあります。これらのポリシーが曖昧または不明確だと思われる場合は、お問い合わせください：

## トーナメントAPIポリシー

1. **公平でバランスの取れたマッチメイキングシステムを確保する責任があります**

2. **プロジェクトのすべての機能は、すべてのトーナメント参加者に無料で利用可能でなければなりません**

3. **トーナメントには、チームサイズに関係なく（1v1、3v3、5v5など）、最低20人のアクティブな参加者が必要です**

4. **チームは対戦相手と直接対戦することでトーナメントを進める必要があります。従来型のトーナメント（エリミネーション、ラウンドロビンなど）でなければならず、直接チャレンジやラダーシステムは許可されません**

5. **賭け、ベッティング、ギャンブル、または名目上の参加費以外の実際のお金の使用に関連するものは許可されません。例外はありません**

6. **金銭的価値を持つカスタム通貨は厳格に禁止されています。参加費や買い込み金は、参加者に明確さを提供するために法定通貨で表示する必要があり、トーナメント終了時に順位に基づいて勝利チーム間で配分される必要があります**
   - 法定通貨とは、政府の規制または法律によって裏付けられた通貨です

## 重要な注意事項

お金が関わる場合、私たちは非常に真剣に対応します。トーナメントAPIに関して相当量の怪しい行動を見てきたため、これは私たちが理解を示す可能性が低い分野です。コミュニティトーナメントがプレイヤーにもたらした圧倒的な利益を理解していますが、これらのポリシーからの逸脱は厳格に処理される可能性があります。これらのポリシーと矛盾する可能性のある機能を実装する前に、質問がある場合はお問い合わせください。

プロジェクトを登録する前に、サードパーティ開発者ツールを使用する際の要件に従うことに同意するかどうかを確認する必要があります。
