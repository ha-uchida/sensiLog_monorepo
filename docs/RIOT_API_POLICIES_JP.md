# Riot Games API 一般ポリシー

これらを十分にお読みください。

すべてのプロジェクトはこれらのポリシーの対象となります。プロジェクトオーナーはこれらのポリシーを読み、理解することが求められます。これらのポリシーに従わない場合、Riot Games APIへのアクセスが無期限に取り消される可能性があります。

Riot Games APIおよびその他の開発者ツールを使用する際、利用規約および法的事項にいくつかの条件を定めています。これらは非常に読みにくい内容であることを理解していますので、最もよく出てくる重要なポイントの概要をまとめました。（ただし、これはすべてを網羅したものではなく、利用規約を読んで同意する必要があることに変わりはありません）

目標は、League of Legendsコミュニティを豊かにし、より良いプレイヤー体験を提供することです。あなたのプロジェクトがネガティブなプレイヤー体験につながる可能性がある場合は、手遅れになる前に私たちに相談してください！柔軟に対応できることもありますが、厳格な姿勢を取ることもあります。

## してはいけないこと

- 法律を破ること
- 公式ロゴを使用すること
- あなたのプロジェクトがRiot Gamesとのパートナーシップまたは承認を受けていると言及すること
- APIキーを適切に保護していないプロジェクトを公開すること
- コミュニティがアクセス可能なプロジェクトを実行するために開発用または暫定APIキーを使用すること（開発用および暫定APIキーは、プロジェクトが公開される前に私たちがレビューできるプロトタイプを作成するために使用されるべきです）
- 複数のプロジェクトを実行するために本番APIキーを使用すること（各プロジェクトは個別に申請し、レビューを受ける必要があります）
- ゲームの整合性を損なったり、プレイヤーに不公平な利点を与えること
- アプリに対して料金を請求したり、特定のユーザーに全体または一部への独占的なアクセスを提供すること
- 最近のパフォーマンスを含む任意の指標に基づいてプレイヤーを非難すること。プレイヤーを称賛したり栄誉を与えることはできますが、プレイヤーのネガティブな先入観につながる可能性のある想定は許可していません
- 他のプレイヤーを報告または評価するための代替チャネルを提供すること
- ランクリーグなどの公式スキルランキングシステムの代替を作成すること（禁止される代替にはMMRまたはELO計算機が含まれます）
- サードパーティツールに含まれていない、リーグチャットなどの他のLeague of Legendsシステムに接続する方法を利用すること（公式の例外で特に記載されている場合を除く）
- 提供されたRiot APIエンドポイントおよびその他の文書化されたサードパーティ開発者ツール以外の、文書化されていないエンドポイントまたはその他のソースからデータをスクレイピングすること（公式の例外で特に記載されている場合を除く）
- ゲーム内外を問わず、ネイティブのLeague of LegendsまたはRiot Gamesのブランディングとデザインのように見えるツールやコンポーネントを構築または設計すること

グレーゾーンに該当する可能性があるアイデアがある場合は、プロジェクトの申請でお気軽にお問い合わせください。プロジェクトの説明と目標を含め、申請内のApp Noteとして質問を投稿してください。フィードバックを提供し、プレイヤー体験を保護するために設けられたポリシーと競合することなく目標を達成するために協力させていただきます。

## 推奨事項

これらは必須要件ではありませんが、私たちが喜んで見るプロジェクトの種類です。

- 既成概念にとらわれない考え方をする
- 私たちを驚かせる、それは奨励されています
- プレイヤーと友達をつなぐ方法を考える
- プレイヤーが自分のゲームプレイを評価し、改善する方法を考える
- ゲームのアートアセットを自由に使用する（ただし、公式ロゴは除く）

前述のとおり、これは包括的なリストではありませんので、利用規約および法的事項もすべて必ずお読みください！

プロジェクトを登録する前に、サードパーティ開発者ツールを使用する際の要件に従うことに同意するかどうかを確認する必要があります。
